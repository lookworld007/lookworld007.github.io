一、网络安全概念及规范

1.1 网络安全定义

1.1.1 网络安全定义

  • 广义：cyber security-网络空间安全。由信息基础设备和网络组成的网络空间安全。
    • 主要由国家从法律、制度、规章和流程等方面指导各政府机构、组织、各行业乃至个人，共同构建国家层面的网络安全。
  • 狭义：network security-网络及网络中传递的信息安全。保障网络中信息及数据的CIA:机密性（confidentiality）、完整性（integrity）、可用性（availability）能力。
    • 包括网络安全厂商针对各类行业客户推出的网络安全设备及方案，主要用于保障各类网络的安全运行。

1.1.2 网络安全发展历程

  • 通信安全时期：通过密码技术解决保密问题，保证数据的保密性和完整性（媒介的安全密封，物理地址安全）
  • 信息安全时期：确保计算机中的软、硬件及信息在处理、存储、传输中的保密性、完整性和可用性（保障CIA)
    • 完整性：确保信息在传输过程中不被篡改。如果被篡改，信息的接收方则可以识别。
    • 保密性：确保信息只能被授权的人员获取和使用。数据即使被攻击者窃取，也不能读出正确的信息。
    • 可用性：确保被授权人员在需要时可以获取和使用相关的信息资产。
  • 信息保障时期：强调信息的保密性、完整性、可用性、可控性的信息安全阶段（体系化：从业务、安全体系、管理三个方面构建信息保障体系）
    • 可控性：对信息和信息系统实施安全监控管理，防止非法利用信息和信息系统。
    • 不可否认性：防止信息源用户对发出的信息事后不承认，或者用户接收到信息之后不认账。
    • 从业务入手：不同业务流量有不同的风险点和防御方式
    • 从安全体系入手：通过更多的技术手段把安全管理和技术防护联系起来，主动防御而不是被动保护。
    • 从管理入手：培养安全管理人才建立安全管理制度
  • 网络空间安全时期：信息安全转化为从整体角度考虑其体系建设的信息保障阶段（全域安全）
    • 目标是包含设施、数据、用户、操作在内的整个网络空间的系统安全
    • 国家出台网络安全等级保护等相关标准和规范，引导企业建立信息安全管理体系

1.1.3 建设网络安全的意义

  • 重要性：网络安全对国家安全至关重要
  • 合规性：企业信息安全管理体系建设需要符合网络安全政策要求
  • 效益性：企业建设网络安全可以减少遭受网络攻击后造成的损失

1.2 网络安全常见威胁

  • 黑客
  • 漏洞（脆弱性）
    • 是指计算机系统在硬件、软件和协议的具体事项或系统安全策略上存在缺陷和不足
  • 勒索攻击
    • 入侵–扩散–盗窃-勒索
  • 信息泄露
  • DDos攻击
    • 分布式拒绝服务攻击，使目标服务器的网络或系统资源耗尽，无法为正常用户提供服务
  • 针对供应链的威胁和攻击
    • 将后门组件植入到供应商的核心服务组件包中并提供给官方，官方更新安装包，客户端下载之后植入后门

1.2.1 深度分析现状

动机：炫耀，利益–黑产–信息、数据，政治、商业、国家等

攻击方：黑客（个人、组织、国家），对手、员工等

方式：DDos、sql注入、脚本、0day、病毒、入侵、木马、勒索等

1.2.2 网络攻击模型

1、洛克希德-马丁公司提出”网络杀伤链模型（kill chain)“

1 情报收集–2 工具准备–3 载荷投递– 4 漏洞利用– 5 释放载荷–6 建立通道–7目标达成，后门

1 reconnaissance–2 weaponization–3 delivery–4 exploitation–5

2、攻击树模型

用AND-OR形式的树结构对目标对象进行网络安全威胁分析。

3、MITRE ATT&CK模型

  • 初始访问–initial
  • 执行–excution
  • 持久化–persistence
  • 特权提升–privilege escalation
  • 躲避防御–defense evasion
  • 凭据访问–credential access
  • 发现–discovery
  • 横向移动–lateral movement
  • 收集–colection
  • 指挥和控制–command and control
  • 外泄–exfiltration
  • 影响–impact

1.3 网络安全发展趋势

  • Gartner预测趋势
  • 国家出台相关法律法规（网络安全法、数据安全法、个人信息保护法、关键基础设施保护条例等）
  • 网络安全态势感知（高级威胁检测、全网安全态势感知、快速安全响应）
    • 定义：是一种基于环境动态地、整体地洞悉安全风险的能力，利用数据融合、数据挖掘、智能分析和可视化等技术，直观显示网络环境的实时安全状况，为网络安全提供技术支撑。
    • 功能：
      • 安全数据分析和结果展示：利用数据挖掘和智能分析等技术，提取系统安全特征和指标，发现网络安全风险，汇总成有价值的情报，并将网络安全风险通过可视化技术直观的展示出来
      • 安全要素采集：采集各类安全设备的海量数据，包括数据流量、各类日志、漏洞、木马和病毒样本等
      • 安全数据处理：对采集到的安全要素数据进行清洗、分类、标准化、关联补齐和添加标签等操作，将标准数据加载到数据存储中。
  • 零信任安全（永不信任、始终验证）

二、网络基础知识

2.1 网络参考模型

2.1.1 数据

应用会产生数据，数据是各种信息的载体，是对客观事物的性质、状态以及相互关系等进行记载的物理符号或者这些符号的组合。数据可以是符号、文字、数字、语音、图像和视频等。

2.1.2 OSI七层参考模型

OSI/RM参考模型（open system interconnection reference model），是由国际标准化组织ISO(international organization for standardization)于1984年发布的用于开放网络互联的模型。由七个层级组层。

2.1.3 TCP/IP参考模型

2.1.4 应用层常见协议

2.1.4.1 FTP

文件传输协议（file transfer protocol),客户端/服务器模式：C/S，多通道协议（需要多个端口表示多个通道），建立两个通道，控制信道和数据信道

1、控制信道：由客户端发起

server：21端口，client：随机端口

2、数据信道

主动模式–服务器发起，server:20端口，client：随机端口

被动魔兽–客户端发起，服务器接收，server：随机端口，client：随机端口

2.1.4.2 HTTP

超文本传输协议（hyper text transfer protocol），传递网页（html,脚本等），c/s ,b/s模式，可以上传数据到服务器，也可以下载服务器数据。数据是明文。

2.1.4.3 HTTPS

hyper text transfer protocol secure,加密传递网页信息，实现的功能包括：加密传递、身份验证、完整验证

2.1.4.4 DNS

域名解析（domain name system）将域名解析为IP地址

2.1.5 传输层常见协议

段–segment

封装：逐层在原来的数据上面添加额外的信息，添加在前面叫头部信息，添加在后面叫尾部信息。

解封装：逐层去掉封装信息的过程就是解封装。

2.1.5.1 UDP

用户数据报协议（user datagram protocol)，8字节，标识上层协议，数据分段，协议号为17

2.1.5.2 TCP

传输控制协议（transmission control protocol）20字节，最大60字节。

1、3次握手，面向连接、可靠传递

2、4次挥手、释放连接

3、确认机制

4、流量控制

2.1.6 网络层常见协议

包-packet

2.1.6.1 IP

Internet protocol，源IP地址和目的IP地址，封装在IP报文里，并且不会发生改变。

数据转发，查询路由表进行转发。

主机的IP地址配置在网卡上，网络设备的IP地址配置在网口上

路由表：里面的信息可以由管理员手工进行配置，指示网络设备、网络里面有哪些网段信息，如何发送，也可以由动态路由生成和接口自动生成

2.1.7 数据链路层

数据链路层位于网络层和物理层之间

数据链路层向网络层提供“段内通信”

负责组帧、物理编址、差错控制等功能

常见的数据链路层协议有以太网、pppoe\ppp等

2.2 常见网络设备

2.2.1 交换机

通常用来组建局域网，连接各种终端设备，根据mac地址表转发数据帧。

2.2.2 路由器

用来连接不同的网络，能够在不同的网络间转发数据

2.2.3 防火墙

主要用于网络边界，本质是控制

防火墙的发展历史：包过滤防火墙、代理防火墙、状态检测防火墙、下一代防火墙

防火墙功能：

  • 只要用于保护一个网络免受来自另外一个网络的入侵和攻击。
  • 隔离不同安全级别的网络
  • 实现不同安全级别的网络之间的访问控制（安全策略）
  • 用户身份认证
  • 实现远程接认证
  • 实现数据加密及虚拟专用网
  • 执行网络地址转换
  • 其他安全功能

2.2.4 wlan设备

WLAN设备由（ac+ap）组成

其中ap由胖ap（FAT AP）和瘦AP(FIT AP)组成，FAT ap本身具有管理功能，FIT ap由AC进行控制

三、常见网络安全威胁及防范

3.1 企业网络网络安全威胁概况

3.1.1 内部和外部的安全威胁

  • 外部威胁：来自企业外部的安全威胁，例如ddos、病毒、木马、蠕虫等网络入侵，网络扫描、垃圾邮件、钓鱼邮件、，针对web服务器的攻击等，访客通过内部网络产生的威胁
  • 内部威胁：网络结构不可靠、网络未隔离，终端存在漏洞、员工行为不受控，信息安全违规操作、信息泄露、恶意员工、权限管理混乱，非法接入等

3.2 通信网络安全需求与方案

3.2.1 网络架构可靠性

通信线路、关键网络设备和关键计算设备的硬件冗余

3.2.2 区域隔离

防火墙通过将所连接的不同网络分隔在不同安全区域隔离内网网

3.2.3 信息保密性

使用VPN技术在internet上构建安全可靠的传输通道，可信验证

3.3 区域边界安全威胁与防护

3.3.1 DDos攻击

DDos攻击是指攻击者通过控制大量的僵尸主机，向攻击目标发送大量攻击报文，导致被攻击目标所在网络的链路拥塞，系统资源耗尽，从而无法向正常用户提供服务。

分类：

  • TCP Flood
  • UDP flood
  • icmp flood
  • HTTP flood
  • gre flood等

防御方式：

防火墙和anti-ddos设备

源认证

限流

3.3.2 单包攻击

  • 扫描型攻击

IPscan 利用IP地址扫描工具探测目标地址，确定目标系统是否存在

  • 畸形报文攻击

smurf攻击：发送icmp请求，该请求包的目的地址设置为受害网络的广播地址，源地址为服务器地址。该网络的所有主机都回应此ICMP请求，回应报文全部发往服务器，导致服务器不能正常提供服务。

  • 特殊报文控制类攻击

tracert报文攻击：利用tracert报文探测网络结构

  • 防御方式：防火墙配置单包防御

3.3.3 用户行为不受控

防火墙的内容过滤功能管控用户的上网行为

内容安全过滤：

  • URL(uniform resource locator）过滤可以对员工访问的URL进行控制，允许或者禁止用户访问某些网站或者资源，达到规范用户上网行为的目的。
  • DNS过滤在域名解析阶段进行控制，防止员工随意访问非法或者恶意的网站，带来病毒、木马和蠕虫等威胁攻击
  • 文件过滤通过阻断特定类型的文件传输，可以降低内部网络执行恶意代码和感染病毒的风险，还可以防止员工将公司机密文件泄露到互联网上
  • 内容过滤包含文件内容过滤和应用内容过滤。文件内容过滤是对用户上传和下载的文件内容中的关键字进行过滤。管理员可以控制对哪些应用传输的文件以及那种类型的文件内容过滤。应用内容过滤是应用协议中包含的关键字进行过滤。针对不同的应用，设备过滤的内容不同
  • 邮件过滤：通过检查发件人和收件人的邮箱地址，附件大小个附件个数来实现过滤
  • 应用行为控制功能用来对用户的http行为和ftp行为进行精确的控制

3.3.4 外部网络入侵行为

防火墙、IPS设备上具备入侵防御功能模块

3.5 管理中心安全需求与方案

3.5.1 管理员权限管控

对于可能发生的企业员工的信息安全风险行为，可以从技术和管理两方面进行

  • 技术方面
    • 更严密的权限管理：对于不同级别的企业员工设置不同权限的账号，特别是对运维的权限要遵循最小授权的原则，比如使用UMA统一运维审计对管理员的运维权限进行管控，并监控管理员行为
    • 更可靠的备份机制：对于已经造成生产环境和数据破坏的情况，可以快速恢复，尽量减少损失
  • 管理方面
    • 在企业内部经常进行信息安全案例宣传，提高员工安全意识
    • 对于高安全需求的区域，可以使用门禁系统
    • 关注员工工作生活状态，及时进行心理辅导，防止员工因心理问题造成的信息安全风险行为。
